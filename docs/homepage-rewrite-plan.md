# MiracleFarms 首页改写方案

## 目标

按照 2026-03-14 讨论共识，将首页从「按内容类型分栏」重构为「先讲清站点定位 → 再给阅读入口 → 再给方法说明」的结构。

---

## 核心原则

1. **首屏先回答“这是做什么的”**  
   - 不是先讲世界观，而是先明确“AI Infra 公开研究写作站”

2. **每个内容入口都要回答“为什么值得点”**  
   - 卡片必须有一句短摘要（1-2 行）
   - 说明解决了什么问题/判断分歧/值得重新看的理由

3. **Farms 解释下沉到 About**  
   - 首页只保留隐约感知，不展开叙事

4. **减少宏大愿景篇幅**  
   - 聚焦：研究什么、怎么研究、最近写了什么、为什么值得读

---

## 目标结构（从顶部到底部）

### 1. Hero 首屏

**当前问题**：信息面板太重，叙事偏多

**目标**：
- 第 1 行：品牌名
- 第 2 行：一句站点定义（AI Infra 公开研究日志）
- 第 3-4 行：关注领域 + 农场隐喻（轻轻带过）
- 按钮：3 个 CTA

**文案方向**（采用 A+C 中间态）：

```
MiracleFarms
公开记录 AI Infrastructure 的演进、实验与判断。

研究推理系统、agent runtime、memory、evaluation 与工程实践。
像经营农场一样，持续搭建、观察、迭代。

按钮：
  从这里开始  ·  阅读最新  ·  进入 GitHub
```

---

### 2. 从这里开始（三入口卡片）

**新增模块**，替代当前的「内容入口」区块

**设计**：3 张水平卡片

| 卡片 | 标题 | 短摘要（why 值得点） | 按钮 |
|------|------|----------------------|------|
| 1 | 认识 MiracleFarms | 为什么要用“农场”来写 AI Infrastructure | 了解这个站点 |
| 2 | 看最新内容 | 快速浏览最近一篇简报，了解我们最近在追踪什么 | 阅读最新 |
| 3 | 深度阅读 | 从一篇代表性长文开始，进入推理系统与工程细节 | 开始深读 |

---

### 3. 最近更新

**当前问题**：Briefs / Essays / Foundations 分开展示，重复感强

**目标**：
- 只保留一个「最近更新」区块
- 混排最新内容（Brief + Essay + Foundation）
- 每张卡片必须带「一句为什么值得读」
- 用 tag 区分类型

**卡片格式**：

```
[标签] 标题
一句话摘要（为什么值得点）
日期
```

---

### 4. Topic Map（内容关注地图）

**当前状态**：已有雏形，基本可用

**微调**：
- 副标题改为「MiracleFarms 长期关注的几块土壤层」
- 底部加一句品牌化文案：「我们不把这些主题当作热点分类，而是把它们当作需要长期耕作的地块。」

---

### 5. How We Work（工作方法）

**新增模块**，建立信任

**文案**：

```
MiracleFarms 是一个 agent-assisted 的公开写作站点。
我们用 agent 帮助检索、整理、比较与草拟，
但判断、取舍与最终发布由人完成。

四步小卡片：
  Research   ·   Drafting   ·   Verification   ·   Judgment
```

---

### 6. About

**当前问题**：在首页展开太多背景故事

**目标**：缩短，只解释名字来源，不展开完整世界观

---

### 7. Footer Tagline

**当前**：无明确 tagline

**目标**：

```
Less hype, more systems.
公开生长，而不是一次性完成。
```

- 每张卡片加一句短摘要

### Step 4：微调 Topic Map

- 改副标题
- 加底部品牌文案

### Step 5：新增 How We Work

- 在 About 之前插入

### Step 6：简化 About

- 缩短文案

### Step 7：加 Footer Tagline

- 收束整个页面

## 验收标准

1. 首屏 3 秒内能说清楚“这是什么站”
2. 每个内容卡片有一句“为什么值得读”
3. 首页不再有宏大愿景叙事
4. Farms 隐喻只在 About 展开
5. 页面有清晰的信息流：定位 → 入口 → 内容 → 方法 → 关于
